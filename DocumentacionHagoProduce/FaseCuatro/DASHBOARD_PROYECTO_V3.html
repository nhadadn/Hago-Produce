⚠️ Gestión de Riesgos
Matriz de riesgo Sprint 4 — Impacto vs Probabilidad
🔴 R01 — Tests OpenAI en CI
Tests de analyzeIntent llaman API real de OpenAI. Sin mock → CI falla en cada push.
Mitigación: jest.setup.ts con mock global de openai-client. Prioridad máxima en S4-P01-A.
🔴 R02 — Design Tokens rompen shadcn
Reemplazar variables CSS puede romper componentes shadcn existentes.
Mitigación: Migración incremental. Nuevos tokens como extensión, no reemplazo. Test visual en cada componente.
🟡 R03 — Chart.js bundle size
Chart.js puede aumentar bundle >200KB, afectando LCP.
Mitigación: Dynamic import + tree shaking. Solo importar charts usados.
🟡 R04 — ZIP timeout >10 facturas
Generación de PDF para múltiples facturas puede timeout en >10 items.
Mitigación: Chunks de 5, progress bar, timeout de 60s, máximo 50 facturas.
🟡 R05 — Secrets en Staging
Variables de entorno de staging pueden exponer secrets de producción.
Mitigación: Variables separadas por environment. No compartir DB de producción.
🔵 R06 — LCP >2.5s Landing
Imágenes sin optimizar pueden degradar LCP en Landing Page.
Mitigación: next/image obligatorio, WebP format, lazy loading, gradientes CSS en hero.
